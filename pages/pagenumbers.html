Numbering conventions people actually argue about

Front matter in books traditionally uses roman numerals while body pages use arabic digits starting at 1. Court filings sometimes demand consecutive arabic numbers across the entire PDF including exhibits—always read the standing order. Academic theses may want the title page unnumbered but counted; that is a layout decision, not something a quick tool guesses for you, so set the starting index deliberately.

Slide decks exported to PDF often want “Page X of Y” in the corner; static “Page X” is easier and still helps reviewers during markup marathons. Meeting packets benefit from centered footers so two-up printing still shows numbers after folding.

Positioning tips

Bottom-center is the default mental model because printed binders hide lower corners under thumbs. Top-right works when footnotes eat the bottom margin. If you number landscape pages, preview both orientations; what clears the margin in portrait may collide with charts when rotated.

Accessibility and reflow

Screen readers announce page breaks based on structure tags, not the visual digit you drew. Page numbers still help sighted collaborators during phone reviews (“jump to page 37”). Do not rely on numbering alone for legal references—cite section headings too when precision matters.

Downstream tools

After numbering, teams often merge chapters, watermark drafts, or lock distribution copies. If pagination shifts because you inserted a cover page, regenerate numbers instead of hand-editing one page in Illustrator—consistency beats speed here.

Some style guides want the first physical page blank but still counted; others want the table of contents in roman numerals separate from body pagination—plan those segments before you automate numbering across the whole file.

Machine-readable page labels

Tagged PDFs expose page labels to assistive tech separately from the visual digit; most lightweight numbering tools adjust the visual layer only. If you must satisfy WCAG plus court formatting, test with Acrobat’s Read Out Loud and a screen reader after numbering. Jumps in sequence (“… 12, 14”) confuse everyone—fix source merges before you stamp digits on top.

Bates numbering for discovery is a different beast: dedicated legal software increments per page globally across productions; use this tool for informal bundles, not for stamped Bates unless counsel confirms the output meets their spec.

Slip sheets and covers

Inserting a color cover after numbering shifts every footer unless you regenerate. Some teams number only the body PDF and keep covers as a separate file in the binder pocket—old-school, but it avoids re-export churn when marketing refreshes the cover art weekly. If you insist on one merged file, automate numbering last in the pipeline, after merge and rotation, so the count matches what approvers already reviewed.

Conference proceedings sometimes restart page numbers per paper inside one PDF; that pattern is painful to automate blindly—split per paper, number, then merge if you truly need a single upload file.

Odd/even headers in Word often assume facing pages; when you flatten to PDF, verify both sides still read correctly before you bake numbers in.

Regenerate instead of patching a single page by hand—manual edits are where numbering quietly drifts out of sync.
